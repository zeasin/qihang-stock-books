# 启航电商进销存账本 - MVP 功能清单 (V1.0)

## 一、版本目标
- **解决核心痛点**：小卖家算不清利润、库存混乱、对账低效
- **功能边界**：进销存 + 利润核算（不含自动拉单、多店铺合并、高级权限）
- **交付形式**：独立部署（提供一键安装包）或 SaaS 版（单租户）
- **种子用户量**：50~100 家真实电商卖家，验证闭环

## 二、模块及功能详情

### 1. 基础设置
| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 店铺管理 | 支持添加/编辑/删除店铺（淘宝、拼多多、抖店等），记录店铺名称、平台类型 | P0 |
| 商品管理 | 支持手工录入商品（SPU/SKU），包含商品名称、规格、采购价、参考售价 | P0 |
| 初始库存 | 支持导入/手工录入期初库存（商品+数量+成本价） | P0 |
| 成本核算方式 | 固定成本（暂只支持按采购价核算，后续版本再支持加权平均等） | P0 |
| 费用类型预设 | 预设常用费用：平台佣金、广告费、运费、包装费、退款损失；用户可自定义费用项 | P1 |

### 2. 采购管理（进）
| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 采购入库单 | 手工创建采购单（选择商品、数量、采购单价、供应商、入库仓库），保存后自动增加库存 | P0 |
| 采购单列表 | 查看/搜索/编辑/删除采购单 | P0 |
| 供应商管理 | 基础供应商信息维护（名称、联系方式、备注） | P1 |

### 3. 销售管理（销）
| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 销售出库单 | 手工创建销售单（选择商品、数量、销售单价、客户、销售日期），自动减少库存 | P0 |
| 销售单列表 | 查看/搜索/编辑/删除销售单 | P0 |
| 客户管理 | 基础客户信息（名称、联系方式、备注），支持散客 | P1 |
| 销售退货单 | 录入退货商品，退回库存并记录退货损失（可选计入成本） | P1 |

### 4. 库存管理（存）
| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 实时库存查询 | 按商品/SKU查看当前库存数量、成本价、库存金额 | P0 |
| 库存流水 | 查看所有出入库流水（采购入库、销售出库、退货等），支持时间/商品筛选 | P0 |
| 库存预警 | 设置库存下限，低于阈值时在首页高亮提醒 | P2 |
| 盘点单 | 录入实盘数量，系统自动生成盘盈盘亏单，调整库存 | P2 |

### 5. 对账与利润核算（核心）
| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 平台对账单导入 | 支持淘宝、拼多多、抖音电商的对账单Excel导入，自动解析字段（订单号、商品、金额、平台扣费） | P0 |
| 费用手工录入 | 允许手工录入广告费、物流费、包装费、杂费等（支持批量导入） | P0 |
| 订单与销售单匹配 | 将对账单中的订单关联到已录入的销售单（按订单号/商品/时间自动或手工匹配） | P0 |
| 单品利润计算 | 自动计算每个商品的毛利润 = 销售价 - 采购成本 - 分摊费用（按数量或金额分摊） | P0 |
| 订单级利润报表 | 展示每个订单的销售额、成本、费用、利润、利润率 | P0 |
| 店铺汇总利润 | 按店铺汇总时间段内的总销售额、总成本、总费用、净利润 | P0 |
| 月度利润报表 | 按月展示店铺利润趋势，支持导出Excel | P0 |
| 费用分摊规则 | 支持按销售额比例或按订单数量将广告费等分摊到每个订单 | P1 |

### 6. 财务报表
| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 经营利润表 | 收入 - 成本 - 费用 = 净利润（主视觉图表+明细表格） | P0 |
| 商品销售排行 | 按销售额/销量/利润排序，辅助分析爆款与滞销品 | P0 |
| 成本分析 | 采购成本趋势、库存成本占比 | P2 |

### 7. 系统与数据
| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| 用户管理 | 单用户（种子期不支持子账号），后续再扩展 | P0 |
| 数据导入/导出 | 支持商品、采购单、销售单的Excel批量导入/导出 | P0 |
| 数据备份 | 支持手动备份数据库（独立部署优势） | P1 |
| 基础日志 | 记录关键操作（登录、导入、删除），便于追溯 | P2 |

## 三、非功能性要求（MVP）
- **部署方式**：提供 Docker 镜像或一键安装脚本（Windows/Linux），数据本地存储
- **性能**：支持单店铺半年内不超过 5000 个订单，库存 SKU ≤ 2000 个
- **界面风格**：后台管理风格（左右布局），简单直观，无需复杂交互
- **浏览器支持**：Chrome / Edge 最新版本
- **错误处理**：导入对账单时提供清晰的错误提示（列名不匹配、必填项缺失等）

## 四、不包含的功能（明确排除）
- ❌ 自动拉取平台订单（依赖 RPA 或 API）
- ❌ 多店铺智能合并报表（MVP 只支持单店铺，但后台数据结构可扩展）
- ❌ 移动端 App（仅 web 端）
- ❌ 复杂权限控制（RBAC）
- ❌ 生产制造、委外加工等非商贸功能
- ❌ 与外部财务软件对接（如金蝶、用友）

## 五、开发顺序建议（按模块）
1. **基础设置** → 商品、店铺、库存初始
2. **采购入库** → 库存增加
3. **销售出库** → 库存减少，同时生成销售单
4. **库存查询 + 流水** → 验证库存变动正确性
5. **平台对账单导入 + 费用手工录入**
6. **订单匹配 + 利润计算** → 核心价值闭环
7. **利润报表 + 商品排行** → 展示结果
8. **数据导入导出、备份** → 提升易用性

## 六、技术栈参考（灵活选择）
- 后端：Python (Django/Flask) 或 Go (Gin) 或 Java (Spring Boot)
- 前端：Vue3 + Element Plus 或 React + Ant Design
- 数据库：SQLite（初始）/ PostgreSQL（生产）
- 部署：Docker Compose + Nginx
